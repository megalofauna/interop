Button

Enhanced button with activation guardrails — debounce, throttle, and reentrancy protection built in. Drop-in replacement for any native button.

Throttle fires immediately then suppresses further activations for the configured window. Useful for physical actions like engine bursts where rapid repeat clicks should be ignored.
Debounce delays activation until the user stops clicking. Useful for search or filter inputs where you want to wait for a pause before firing.
With reentrant: false, a second click while the handler is still running is dropped. Prevents double-submission on slow async operations like docking sequences or cargo transfers.
